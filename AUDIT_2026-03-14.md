## 🔴 CRITICAL FINDINGS (15)

### Findings That Likely Affect Current Simulation Results

**1. Immunosuppression timers double-decremented during spawning season**
- *Source:* spawning.py L170 + model.py L1540/L2533
- `spawning_step()` decrements timers, then the main loop decrements again same day
- 28-day immunosuppression → ~15 effective days during spawning season (~257 days/yr)
- **Impact: Disease-reproduction coupling is mis-calibrated**
- Fix: Remove decrement from `spawning_step()` (1-line fix)

**2. Ghost cascade induction across years**
- *Source:* spawning.py L346, L382–404
- `last_spawn_day` stores day-of-year, not sim_day. Not reset between seasons.
- When same DOY recurs next year, agents falsely appear as "just spawned" → trigger cascade induction
- Fix: Reset `last_spawn_day` in `reset_spawning_season()` or store sim_day

**3. Population can exceed K with pre_filtered=True**
- *Source:* model.py L872–876
- Spatial sim's `settle_daily_cohorts` doesn't cap `n_recruits` to `available_slots`
- Between spawning-time BH and settlement-time placement, other cohorts/releases can fill slots
- Fix: Add `n_recruits = min(n_recruits, available_slots)` in pre_filtered branch

**4. `target_mean_c` defaults to 0.08, not the documented 0.02**
- *Source:* genetics.py L218 (`initialize_genotypes_three_trait`)
- Recovery trait 4× too high if any caller uses the default
- `genotypes_from_trait_targets()` correctly defaults to 0.02 — inconsistency
- **Needs verification: does model.py pass explicitly?**

- **Needs verification: does the model mutate the network object?**

## 🟠 HIGH FINDINGS (38) — Top Impact

### Affect Simulation Dynamics

**H-1. Growth noise 27× too small in daily mode**
- *Source:* model.py L536–540
- Daily σ compounds to ~0.0055/yr vs annual σ=0.15. Growth is effectively deterministic.
- Affects size-at-age heterogeneity → reproductive eligibility → population dynamics

**H-2. `size_susceptibility` attenuated 100× from Eisenlord**
- *Source:* disease.py L129–138
- Divides by `SIGMA_L=100`. 500mm vs 300mm star = 1.04× instead of 66×
- May be intentional (ABM stability) but docstring is misleading

**H-3. Hardcoded 10K larval cap throttles recruitment**
- *Source:* spawning.py L761–767
- `max_larvae_per_cohort = 10000` — test concern bleeding into production
- Comment: "for computational efficiency (especially in tests)"

**H-4. SRS pair selection always picks largest individuals**
- *Source:* spawning.py L738–742
- `argsort(size)[::-1]` = deterministic assortative mating, not sweepstakes lottery
- Small females never breed

**H-5. No fertilization kinetics — 1 male fertilizes unlimited females**
- *Source:* spawning.py L728–733
- 50 females + 1 male → 500M eggs fertilized. `gamma_fert` exists in config but unused.
- Artificially accelerates recovery in male-limited (post-crash) populations

**H-6. V_A formula is 4× inflated**
- *Source:* genetics.py L419
- Uses `2e²pq` but correct formula is `e²pq/2` (allele substitution α = e/2)
- Affects diagnostics only, not simulation dynamics

**H-7. Hardcoded fecundity params diverge from config**
- *Source:* spawning.py L740–745
- `F0=1e7, L_ref=500, fecundity_exp=2.5` hardcoded, not read from PopulationSection

**H-8. Two divergent SRS implementations**
- *Source:* spawning.py L730–745 vs reproduction.py
- Single-node: deterministic size-ranked. Multi-node: weighted random SRS.

## 🎯 Triage: What Actually Affects Current Calibration Runs

These findings could be actively distorting W229-W248 and all prior sweep results:

1. **Immunosuppression double-decrement** (Spawning C1) — disease-reproduction coupling wrong
2. **Ghost cascade induction** (Spawning C2) — false spawning triggers every year
3. **Population can exceed K** (Core C-3) — density dependence bypassed
4. **Growth noise 27× too small** (Core H-3) — no size heterogeneity
5. **10K larval cap** (Spawning H-3) — recruitment throttled by arbitrary limit
6. **No fertilization kinetics** (Spawning H-5) — unlimited male fertility accelerates recovery
7. **Assortative mating by size** (Spawning H-4) — not SRS

These are "design choices that compound" — individually each might be small, but together they shape the dynamics the calibration is trying to fit.

### What's Probably Fine

- **Shared network across seeds** (Expt C-1) — needs verification but spatial sim likely creates internal copies
- **target_mean_c = 0.08** (Gen C-1) — likely passed explicitly by model.py callers
- **build_network defaults** (Spatial C-1) — only affects tests, not production runs via calibration_runner
- **V_A 4× inflated** (Gen H-1) — diagnostics only, doesn't affect simulation

### What Should Be Fixed Before Next Sweep

Priority order:
1. Immunosuppression double-decrement (1-line fix)
2. Ghost cascade induction (1-line fix in reset)
3. Population K overshoot cap (1-line fix)
4. Growth noise scaling (formula fix)
5. Document/decide on fertilization kinetics and larval cap

### What Can Wait

- Dead code cleanup (H-19 through H-23)
- Test improvements (H-15, H-26, H-29, H-35)
- Performance optimizations (H-33, H-27, H-30)
- Config architecture (tau_max duplication, ANNUAL_SURVIVAL, build_network defaults)
- Docstring corrections (C-13, H-2, H-17, H-38)
